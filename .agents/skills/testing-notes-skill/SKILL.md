# Testing Notes Skill

Use this skill to turn current behavior into a practical test plan.

## Core goals

1. Cover highest-risk behavior first.
2. Include edge cases and error paths.
3. Keep tests concrete and executable.

## Workflow

1. Identify critical behaviors and seams.
2. Propose happy path and negative path tests.
3. Suggest mocks/stubs for external boundaries.
4. Add assertions that verify behavior, not implementation trivia.

## Output structure

Prefer:

- Priority test cases
- Edge and failure scenarios
- Mocking strategy
- Coverage gaps
- Source references
