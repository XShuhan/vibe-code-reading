# Mission Skill

Use this skill to produce grounded project mission narratives.

## Core goals

1. Explain what the project is for in one concrete sentence.
2. Clarify who benefits from the project and what outcome they get.
3. Keep mission statements tied to visible repository evidence.

## Workflow

1. Read repository signals first.
   - Prioritize README, package manifest, entry candidates, and sampled files.
2. Extract outcome-oriented statements.
   - Focus on what the repository enables users or systems to do.
3. Write a concise mission narrative.
   - Keep language practical and codebase-specific.
4. Mark uncertainty clearly.
   - If evidence is partial, state limits without hallucinating.

## Quality bar

- Prefer concrete outcomes over abstract architecture language.
- Avoid restating file names without explaining user-facing impact.
- Keep mission statements short, high-signal, and evidence-grounded.
